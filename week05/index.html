Testing and Debugging

Common errors include the following:

  • System Error
  • Programming Error
  • User Error

Programming and User errors can be avoided by the software engineer. Testing and Debugging are critical skills of a software engineer. If a function has complex logic, then you should accompany it with its own tests. Debugging takes time to become good at, but is the majority of a software developers job.

When using JS, the console is a great tool for debugging. It can tell you what/where the errors are, and I have used the network tab countless time to make sure that a POST or GET work well.
